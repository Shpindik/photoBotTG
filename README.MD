# PhotoBotTG — Телеграм-бот для сбора обратной связи

Этот бот предназначен для регистрации проблем и сбора контактов пользователей в привязке к определенным локациям. При возникновении проблемы бот уведомляет администратора и сохраняет данные в базу данных PostgreSQL.

## 🛠 Технологии
- **Язык:** Python 3.10+
- **Библиотеки:** aiogram 3.x, asyncpg, psycopg2
- **База данных:** PostgreSQL 13
- **Развертывание:** Docker & Docker Compose

## 🚀 Быстрый старт с Docker

Это самый простой способ запустить проект:

1.  **Настройте переменные окружения:**
    Создайте файл `.env` в корне проекта и заполните его по примеру:
    ```env
    BOT_TOKEN=ваш_токен_бота
    ADMIN_ID=ваш_телеграм_id  # для уведомлений о проблемах
    
    POSTGRES_DB=photo_bot
    POSTGRES_USER=postgres
    POSTGRES_PASSWORD=your_secure_password
    DB_HOST=db
    DB_PORT=5432
    ```

2.  **Запустите контейнеры:**
    ```bash
    docker-compose up -d --build
    ```

Проект автоматически создаст базу данных (используя `init.sql`), установит все зависимости и запустит бота.


## 📂 Структура проекта
- `bot/main.py` — Точка входа, инициализация бота и обработка сигналов.
- `bot/handlers.py` — Логика обработки команд и сообщений.
- `bot/db.py` — Взаимодействие с PostgreSQL через `asyncpg`.
- `bot/kb.py` — Определение клавиатур (Reply/Inline).
- `bot/states.py` — Состояния FSM (Finite State Machine).
- `bot/dict.py` — Текстовые константы и сообщения.
- `logs/` — Директория для логов ошибок.

## 📝 Основные функции
- **Привязка к локации:** Бот может принимать параметр локации (например, через ссылку `t.me/bot?start=LOCO1`) или запрашивать её вручную.
- **Сбор контактов:** Бот запрашивает номер телефона после подтверждения локации (кнопкой или текстом).
- **Уведомление админа:** Каждая отправленная проблема немедленно пересылается администратору (`ADMIN_ID`).
- **Логирование:** Ошибки сохраняются в `logs/bot_errors.log`.
